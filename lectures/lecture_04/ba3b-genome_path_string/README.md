# Genome Path String (BA3B)

## Задача

Восстановить строку по её геномному пути (последовательности перекрывающихся k-меров).

**Вход:** последовательность k-меров `Pattern₁, ..., Patternₙ`, где последние k−1 символов `Patternᵢ` совпадают с первыми k−1 символами `Patternᵢ₊₁`
**Выход:** строка `Text` длины k+n−1, составленная из данных k-меров

Формат входного файла Rosalind:
```text
kmer1
kmer2
...
kmern
```

Программа поддерживает два режима:
- Rosalind-style аргументы: `kmer1 kmer2 ...`
- путь к входному файлу в формате Rosalind

Результат печатается в `stdout` и сохраняется в файл:
- при `--output <path>` — в указанный путь
- при запуске от входного файла без `--output` — в `<input>.out`
- при запуске от аргументов или sample dataset без `--output` — в `genome_path_string_output.txt`

**Пример:**
```
Вход:
ACCGA
CCGAA
CGAAG
GAAGC
AAGCT

Вывод: ACCGAAGCT
```

## Теория

### Геномный путь

Строка `Text` длины n может быть разбита на последовательность перекрывающихся k-меров:
```
Text = s₁s₂...sₙ
Pattern₁ = s₁s₂...sₖ
Pattern₂ = s₂s₃...sₖ₊₁
...
Patternₙ₋ₖ₊₁ = sₙ₋ₖ₊₁...sₙ
```

Эта последовательность k-меров называется **геномным путём** (Genome Path) строки `Text`. По сути, это результат операции из BA3A (k-мерная композиция), но **с сохранённым порядком**.

### Ключевое свойство перекрытий

Соседние k-меры в геномном пути перекрываются на (k − 1) символов. Визуально:
```
ACCGA
 CCGAA
  CGAAG
   GAAGC
    AAGCT
─────────
ACCGAAGCT
```

Каждый последующий k-мер добавляет ровно **один новый символ** — свой последний. Это даёт формулу: длина результата = k + (n − 1), где n — количество k-меров.

Формально: `suffix(Patternᵢ) = prefix(Patternᵢ₊₁)`, где суффикс и префикс имеют длину k − 1.

### Биологический смысл

В задаче BA3A мы разобрали строку на k-меры и потеряли порядок. Здесь — обратная операция: если кто-то подсказал нам правильный порядок фрагментов, восстановить исходную строку тривиально.

На практике это соответствует идеальной ситуации: риды уже выстроены вдоль генома (например, если геном секвенирован с помощью нанопор в одном длинном чтении, а затем фрагментирован). Реальная сложность сборки генома — не в склейке, а в **определении порядка** фрагментов. Именно для этого в следующих задачах строятся графы перекрытий (BA3C) и графы де Брёйна (BA3D–BA3E).

### Применения за пределами биологии

- **Сетевые протоколы (TCP/IP):** пакеты данных приходят в произвольном порядке, но каждый содержит порядковый номер. Восстановление исходного потока из упорядоченных перекрывающихся фрагментов — та же задача
- **Восстановление документов:** при сканировании порезанных шредером бумаг — полоски с частичным перекрытием текста склеиваются по совпадающим краям
- **Потоковое аудио/видео:** кодеки используют перекрывающиеся фреймы (overlap-add) для плавного восстановления сигнала
- **Цепочки слов (word chains):** лингвистическая игра, где последние буквы слова совпадают с первыми буквами следующего (ширОКа → ОКАзия → ...)

## Алгоритм

### Псевдокод
```
GenomePathString(Patterns):
    Text ← Patterns[0]
    for i ← 1 to |Patterns| - 1:
        Text ← Text + последний символ Patterns[i]
    return Text
```

Алгоритм линейный: достаточно взять первый k-мер целиком и от каждого следующего добавить только последний символ.

## Реализация на C++

```cpp
std::string solve(const InputData& data) {
    std::string result = data.kmers[0];             // (1)
    for (size_t i = 1; i < data.kmers.size(); ++i) {
        result += data.kmers[i].back();             // (2)
    }
    return result;                                   // (3)
}
```

### Особенности реализации

**1. Инициализация первым k-мером**
```cpp
std::string result = data.kmers[0];
```
- Копируем первый k-мер целиком — это начало результата
- `data.kmers[0]` — доступ к первому элементу вектора за O(1)
- Создаётся копия строки (не ссылка), так как далее `result` будет модифицироваться

**2. `std::string::back()` и `operator+=`**
```cpp
result += data.kmers[i].back();
```
- `back()` — возвращает ссылку на последний символ строки (O(1))
- Эквивалентно `data.kmers[i][data.kmers[i].size() - 1]`, но безопаснее и читаемее
- `operator+=(char)` — добавляет один символ в конец строки
- Амортизированная стоимость O(1) благодаря экспоненциальному росту буфера `std::string`

**3. Возврат строки (move semantics)**
```cpp
return result;
```
- Компилятор применяет NRVO (Named Return Value Optimization) или move-семантику
- Строка **не копируется** при возврате из функции — внутренний буфер перемещается
- В C++17 copy elision гарантирована стандартом для ряда случаев

**4. Валидация перекрытий**
```cpp
for (size_t i = 0; i + 1 < data.kmers.size(); ++i) {
    std::string suffix = data.kmers[i].substr(1);
    std::string prefix = data.kmers[i + 1].substr(0, k - 1);
    if (suffix != prefix)
        throw std::invalid_argument(...);
}
```
- `substr(1)` — подстрока от позиции 1 до конца (последние k−1 символов)
- `substr(0, k-1)` — подстрока от начала длиной k−1 (первые k−1 символов)
- Проверяем, что суффикс каждого k-мера совпадает с префиксом следующего
- `i + 1 < size()` вместо `i < size() - 1` — защита от переполнения `size_t` при пустом векторе


## Сложность

### Время: O(n × k)
- n = количество k-меров
- Основной цикл: n−1 итераций, каждая — O(1) (добавление одного символа)
- Валидация: n−1 сравнений подстрок длины k−1, каждое O(k)
- Итого: O(n × k) с валидацией, O(n) без неё

### Память: O(n + k)
- Результат: строка длины k + n − 1
- Входные данные: n строк по k символов = O(n × k)
