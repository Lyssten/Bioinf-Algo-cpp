# Clump Finding

## Задача

Найти все k-меры, образующие (L, t)-клумбы в геноме.

**Вход:** `Genome`, `k`, `L`, `t`  
**Выход:** все k-меры, которые встречаются ≥ t раз в окне длины L

**Пример:**
```
Genome: CGGACTCGACAGATGTGAAGAAATGTGAAGACTGAGTGAAGAGAAGAGGAAACACGACACGACATTGCGACATAATGTACGAATGTAATGTGCCTATGGC
k=5, L=75, t=4
Вывод: AATGT CGACA GAAGA
```

## Теория

### (L, t)-клумба
k-мер образует **(L, t)-клумбу**, если существует окно длины L в геноме, где этот k-мер встречается как минимум t раз.

**Биологический смысл:** Клумбы часто указывают на регуляторные элементы (например, сайты связывания белков), которые могут быть многократно продублированы в определённой области генома.

### Пример
```
Genome: ...gatcagcataagggtcccTGCAATGCATGACAAGCCTGCAgttgttttac...
                     |______________________________|
                                  L = 25

TGCA встречается 3 раза в окне L=25 → (25,3)-клумба
```

## Алгоритм

### Идея
1. Скользящее окно длины L по геному
2. Для каждого окна считаем частоты всех k-меров
3. Если k-мер встречается ≥ t раз — добавляем в результат

### Псевдокод
```
ClumpFinding(Genome, k, L, t):
    frequentPatterns ← пустое множество
    
    for i ← 0 to |Genome| - L:
        Window ← Genome[i..i+L-1]
        freq ← пустая хэш-таблица
        
        for j ← 0 to L - k:
            pattern ← Window[j..j+k-1]
            freq[pattern] ← freq[pattern] + 1
            if freq[pattern] ≥ t:
                добавить pattern в frequentPatterns
    
    return frequentPatterns
```

## Реализация на C++

```cpp
std::set<std::string> clumpFinding(const std::string& genome, 
                                    size_t k, size_t L, size_t t) {
    std::set<std::string> result;
    
    if (k > genome.size() || L > genome.size())
        return result;

    for (size_t i = 0; i <= genome.size() - L; ++i) {  // (1)
        std::string window = genome.substr(i, L);  // (2)
        std::unordered_map<std::string, int> freq;  // (3)
        
        for (size_t j = 0; j <= L - k; ++j) {
            std::string pattern = window.substr(j, k);
            if (++freq[pattern] >= t)  // (4)
                result.insert(pattern);  // (5)
        }
    }
    return result;
}
```

### Особенности реализации

**1. Внешний цикл по окнам**
```cpp
for (size_t i = 0; i <= genome.size() - L; ++i)
```
- i = начало окна длины L
- Окна перекрываются: сдвиг на 1 позицию

**2. Извлечение окна**
```cpp
std::string window = genome.substr(i, L);
```
- Создаёт копию подстроки длиной L
- **O(L)** по времени и памяти

**3. Локальная хэш-таблица**
```cpp
std::unordered_map<std::string, int> freq;
```
- Создаётся заново для каждого окна
- Считаем частоты только внутри текущего окна

**4. Проверка условия клумбы**
```cpp
if (++freq[pattern] >= t)
```
- Префиксный `++` сначала увеличивает счётчик
- Как только достигли t — добавляем в результат
- Повторные вставки того же паттерна безвредны (set)

**5. `std::set` для результата**
```cpp
result.insert(pattern);
```
- Автоматически удаляет дубликаты
- Один и тот же паттерн может быть найден в нескольких окнах

## Оптимизация

Текущая реализация **O(|Genome| × L × k)** — медленно для больших геномов.

**Возможные улучшения:**

1. **Обновление окна без пересчёта:**
   - При сдвиге окна удаляем уходящий k-мер, добавляем новый
   - Сложность: **O(|Genome| × k)**

2. **Битовое кодирование k-меров:**
   - A=00, C=01, G=10, T=11
   - k-мер = целое число (2k бит)
   - Быстрое хэширование через прямой доступ

## Запуск

```bash
# Без аргументов — тестовые данные
./clump_finding

# Аргументы: <Genome> <k> <L> <t>
./clump_finding "CGGACTCGACAGAT..." 5 75 4
# Вывод: AATGT CGACA GAAGA
```

## Сложность

### Время: O(|Genome| × L × k)

**Обоснование:**
- Внешний цикл (окна): **O(|Genome| - L + 1)** = **O(|Genome|)** итераций
- На каждой итерации:
  - `substr(i, L)` — **O(L)** копирование окна
  - Внутренний цикл: **O(L - k + 1)** = **O(L)** итераций
    - `substr(j, k)` — **O(k)** копирование k-мера
    - Хэширование и вставка — **O(k)**
- Итого: **O(|Genome| × L × k)**

**Пример:** Genome = 1,000,000, L = 500, k = 9  
→ 1,000,000 × 500 × 9 = 4,500,000,000 операций (медленно!)

### Память: O(L × k + m × k)

**Обоснование:**
- `window` — строка длиной L: **O(L)**
- `freq` — хэш-таблица для одного окна: **O((L-k+1) × k)** = **O(L × k)**
- `result` — множество найденных клумб: **O(m × k)** где m = число клумб
- Итого: **O(L × k + m × k)**

**Пример:** L = 500, k = 9, 100 клумб  
→ 500 × 9 + 100 × 9 = 4,500 + 900 = 5,400 байт

### Оптимизация до O(|Genome| × k)

При сдвиге окна на 1:
- Удаляем уходящий k-мер: **O(k)**
- Добавляем новый k-мер: **O(k)**
- Не нужно пересчитывать всё окно!
